Pioneer-sdk для симулятора автономных полетов¶

Для работы с Симулятором автономных полетов необходимо установить флаг simulator = True при создании объекта класса Pioneer():

Pioneer(simulator = 'True')

Примечание

В симуляторе автономных полетов, в отличие от работы реального дрона, доступны следующие методы управления магнитом. Если использовать методы на реальном дроне — вы увидите ошибку NotImplementedError.

Существует два способа включить или выключить магнит:

  • через универсальный метод cargo_set();

  • через отдельные методы cargo_grab() и cargo_release().

cargo_set(grab: bool)¶

Устанавливает состояние магнита.

Параметры:

grab – True — включить магнит (захватить груз); False — выключить магнит (отпустить груз);

Тип результата:

None

Результат:

Выполняет включение или выключение магнита при работе в режиме симулятора. При использовании на реальном дроне вызывает исключение NotImplementedError.

cargo_grab()¶

Включает магнит.

Тип результата:

None

Результат:

Выполняет включение магнита при работе в режиме симулятора. При использовании на реальном дроне вызывает исключение NotImplementedError.

cargo_release()¶

Выключает магнит.

Тип результата:

None

Результат:

Выполняет выключение магнита при работе в режиме симулятора. При использовании на реальном дроне вызывает исключение NotImplementedError.

Pioneer¶

Создание объекта класса:¶

class Pioneer(simulator='False', name='pioneer', ip='192.168.4.1', mavlink_port=8001, connection_method='udpout', device='/dev/serial0', baud=115200, logger=True, log_connection=True)¶
Параметры:

  • simulator – использование библиотеки для симулятора или реального дрона: True - использовать для симулятора; False - для реального дрона;

  • name – путь до Lua-файла;

  • ip – ip-адрес дрона;

  • mavlink_port – порт для обмена MAVLink сообщениями;

  • connection_method – принимает аргументы «udpin»; «udpout»; «serial»;

  • device – указание порта;

  • baud – скорость передачи;

  • logger – логи MAVLink сообщений. True - выводить в консоль; False - не выводить;

  • log_connection – логи соединения. True - выводить в консоль; False - не выводить;

Тип результата:

obj

Результат:

Новый экземпляр класса Pioneer.

Примечание

По умолчанию флаг simulator устанавливается в False. Если планируете использовать библиотеку с реальным дроном - это поле можно не указывать. Если хотите использовать библиотеку с симулятором - укажите simulator = True.

Описание методов класса Pioneer:¶

classmethod connected()¶

Возвращает текущее состояние подключения.

Тип результата:

bool

Результат:

Возвращает False, если соединение потеряно (в течении секунды нет сообщений от дрона), возвращает True, если соединение стабильное.

classmethod close_connection()¶

Закрывает MAVLink соединение с дроном.

Примечание:

Метод поддерживается с версии pioneer_sdk 0.4.3;

classmethod reboot_board()¶

Выполняет перезагрузку дрона.

Тип результата:

bool

Результат:

Возвращает True, если результат „ACCEPTED“ или „DENIED“;
Возвращает False, если результат „SEND_TIMEOUT“, „TEMPORARILY_REJECTED“, „UNSUPPORTED“, „FAILED“, „CANCELLED“;

Подробнее:

в пункте Pioneer.mav_result.
